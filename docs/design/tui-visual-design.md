# MMI TUI 视觉设计

> 2026-06-05 · v0.1-draft（搁置）
> 设计目标：极简、轻量、有呼吸感，类似 Reasonix TUI 风格
>
> **状态：设计稿完成，原型验证搁置。**
> 核心矛盾：Textual 框架不支持真正透明背景且难以精确控制布局；
> Rich 方案可以解决背景和居中，但有全屏闪烁问题（需改用 Live 模式渲染）。
> 后续可考虑用 `rich.live.Live` + 局部刷新方案重试原型。

---

## 1. 设计语言

### 1.1 核心原则

| 原则 | 说明 |
|---|---|
| **轻** | 无厚重边框、无大面积色块、无多余装饰 |
| **薄** | 分割线细（1px）、颜色淡、背景单一 |
| **透** | 大块留白，内容有呼吸感 |
| **设计感** | 分割线不到边、状态压线、符号引导缩进 |

### 1.2 分割线规范

```
       ──────────────────────────
        ↑ 两端各留约 10% 空白
```

- **线型**：纯 `─` 字符重复（不用 Unicode 框线字符）
- **厚度**：单字符高度，视觉上细
- **不到边**：左右两端各留约 10% 的空白（具体比例在 prototype 中调优）
- **打断**：文字/标签压在线上时，线被文字打断，线从文字两侧继续延伸
- **适用场景**：界面分隔、区域标题、快捷键提示行

### 1.3 布局

| 界面 | 排列 | 说明 |
|---|---|---|
| **Session Hub（会话总览）** | 居中 | 标题→列表→快捷键，全部垂直居中 |
| **Chat（聊天主界面）** | 左对齐 | 三段式：信息条 / 对话流 / 编辑区+状态 |

## 2. Session Hub（界面一）

### 2.1 线框图

```
                                           ← 顶部留白约 3 行
                    MMI                          ← 主标题，居中，较大
            Multimodal Intelligence              ← 副标题，居中，较小

       ─────────  Sessions ─────────             ← 标题线，"Sessions" 从线上穿过

              Design Architecture Discussion     12.3   ← 会话列表，居中排列
              Bug Fix Round 8.5                   8.7       每行：标题 + 右侧热度值
              Poetry Corner                       4.2
              Old Project Archive                 1.1

       ─────────  4 sessions ─────────            ← 底部计数线

                                           ← 底部留白
       ───  n new  / search  q quit  ───         ← 快捷键提示，压线
```

### 2.2 交互

| 按键 | 行为 |
|---|---|
| `↑` / `↓` | 选择会话 |
| `Enter` | 进入选中会话的 Chat |
| `n` | 新建会话 |
| `/` | 焦点切到底部搜索区，实时 fuzzy 过滤列表 |
| `q` | 退出 |

### 2.3 搜索逻辑

- 主界面默认展示按热度排序的前 N 条会话
- 搜索是针对**当前列表未展示**的历史会话进行 fuzzy 检索
- 焦点在底部搜索区时输入，结果替换或补充当前列表（防抖 150ms）
- 匹配规则：rapidfuzz partial_ratio 阈值 60
- `Esc` 退出搜索模式，列表恢复默认视图
- 搜索为空时提示"未找到匹配会话"

## 3. Chat（界面二）

### 3.1 三段式布局

```
──────────────────────────────────────────────────   ← 上线（不到边）
 MMI  ·  deepseek-chat  ·  zh-CN                       ↑ 顶部信息条，信息压线

                                                       ← 区域间距
  [你] 帮我看看这个架构设计                             ← 左对齐
                                                       ← 消息间距（空行）
  [MMI] 根据文档，有几个方向可以考虑...
        def process(data):
        ├── cleaned = sanitize(data)                   ← 代码树形线
        │   ├── strip_html()
        │   └── normalize_whitespace()
        ├── result = llm.generate(cleaned)
        └── return post_process(result)
  
        参考：› docs/ARCHITECTURE.md                    ← 引用前缀
        在线：→ https://example.com                     ← 链接前缀

──────────────────────────────────────────────────   ← 编辑区上分割线
 > 输入消息... (/cmd  !bash  $py)                       ← 左对齐，提示文字灰色
──────────────────────────────────────────────────   ← 编辑区下分割线
 12:30  ·  24 turns  ·  87 tokens  ·  Esc to exit     ← 状态行，极小子号
──────────────────────────────────────────────────   ← 底线
```

### 3.2 顶部信息条

压在上分割线上，从左到右：

```
MMI  logo  ·  deepseek-chat  ·  zh-CN  ·  📖 docs
```

- LOGO 用纯文字 "MMI"（不加 ASCII 大图，保持轻量）
- 信息项之间用 ` · ` 中点分隔
- 最后一项可选 `📖 docs` 超链接提示

### 3.3 对话区（核心）

**整体布局**：整个对话区在窗口中**居中**，消息左右留白的比例随窗口宽度自动调整。每条消息作为一个独立块，保持在窗口中间的列里。

**消息格式**：

```
                    [你] 帮我看看这个架构设计

                    [MMI] 根据文档有几个方向...
```

- 标签用 `[` `]` 包裹
- 用户标签用蓝色，Agent 标签用深灰/正文色
- 颜色反差小，不刺眼
- 消息之间用空行分隔（无分割线）
- 代码块也在此居中列内，左对齐

**代码展示**：

```
def pipeline():
├── step1()
│   ├── sub_a()
│   └── sub_b()
└── step2()
```

- 代码块用普通缩进 + 树形线引导
- 树形线字符：`├── ` / `└── ` / `│   `
- 关键词用暗色高亮（#9ece6a 暗绿），字符串略暗

**引用/链接/记忆**：

| 前缀 | 符号 | 用途 | 颜色 |
|---|---|---|---|
| 本地文档 | `›` | `› docs/ARCHITECTURE.md` | 暗蓝紫 |
| 外部链接 | `→` | `→ https://...` | 暗蓝 |
| 记忆引用 | `↳` | `↳ 2026-06-03 讨论了...` | 暗绿 |

**折叠块**：

```
┌─────────────────────────────────────┐
│ [thinking] 考虑用 Router 分类...     │  ← 圆角线框 + 左色条
│ 实际内容...                          │     色条：紫色（thinking）
└─────────────────────────────────────┘     绿色（tool_call）
```

- 用 Unicode 圆角框字符 `┌ ┐ └ ┘ │`
- 左侧竖线着色区分类型
- 点击展开/折叠

### 3.4 编辑区

```
──────────────────────────────────────────────────   ← 编辑区上分割线（固定位置）
 > 输入消息... (/cmd  !bash  $py)                     ← 初始单行，左对齐
 > 第二行...                                          ← 多行时向上扩展
 > 第三行...                                                 
──────────────────────────────────────────────────   ← 编辑区下分割线（被推下）
```

- **初始状态**：单行输入，左对齐，提示文字（#606060）
- **动态扩展**：输入多行时，编辑区向上扩展（下分割线被推下，对话区被压缩）
- 上限不超过窗口高度的 40%，到上限后内部滚动
- 支持：`/cmd` 斜杠命令 / `!bash` shell / `$py` python
- `Enter` 发送 / `Shift+Enter` 换行（常规聊天中）
- 上下两条细线始终跟随编辑区顶部和底部移动

### 3.5 状态行

```
 12:30  ·  24 turns  ·  87 tokens  ·  Esc to exit
```

- 极小子号，颜色最暗（#3b4261）
- 从左到右：时间 / 轮数 / token 数 / 快捷键提示

## 4. 配色方案

### 4.1 设计原则

- **背景不固定** — 背景跟随终端背景色，TUI 本身不设背景色
- **双变体自适应** — 根据终端背景亮度（暗/亮）自动切换颜色变体
- **最小假设** — 只假定终端要么暗色背景要么亮色背景，不假定具体色值

### 4.2 暗色终端变体（默认）

适用于黑色/深色终端背景。

| 角色 | 色值 | 说明 |
|---|---|---|
| 背景 | 透明 | 跟随终端背景 |
| 正文 | `#e0e0e0` | 亮灰白，不纯白（柔和） |
| 用户标签 `[你]` | `#7dcfff` | 浅蓝 |
| Agent 标签 `[MMI]` | `#e0e0e0` | 与正文同色 |
| 分割线 | `#404040` | 暗灰，近似不可见 |
| 压线文字 | `#7aa2f7` | 浅蓝紫 |
| 选中高亮 | `#2ac3de` | 青蓝 |
| 代码/关键词 | `#d4a040` | **暗黄**（与正文形成反差） |
| 代码字符串 | `#8a8a8a` | 暗灰 |
| 树形线字符 | `#606060` | 中灰 |

| 引用前缀 `›` `→` `↳` | `#7aa2f7` | 浅蓝紫 |

## 5. 主题自适应机制

### 5.1 终端背景检测

TUI 启动时通过终端转义序列 `OSC 11 ; ? BEL` 查询终端背景色：

- 如果终端支持响应 → 解析 RGB 值，计算亮度（Luminance）
- 亮度 > 0.5 → 亮色变体；亮度 ≤ 0.5 → 暗色变体
- 如果终端不支持（超时或空响应）→ 默认暗色变体

### 5.2 手动切换

支持用户覆盖自动检测结果：

- Chat 中 `/theme dark` / `/theme light` 即时切换
- 切换时无闪烁（textual CSS 热替换）
- 设置持久化到 `config.toml` 中的 `tui.theme` 字段
- 下次启动优先使用此配置，而非自动检测

## 6. 代码树形线规范

用于代码展示的树形字符序列：

| 深度 | 字符 | 含义 |
|---|---|---|
| 1 | `├── ` | 中间节点 |
| 1 | `└── ` | 最后一个节点 |
| 2 | `│   ├── ` | 嵌套中间节点 |
| 2 | `│   └── ` | 嵌套末节点 |
| 3+ | 同理递推 | — |

这个格式与终端输出风格一致，适合 Python 代码的结构化展示。
